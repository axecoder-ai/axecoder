# 选型记录

## 2a 选型摘要

### 一句话需求回顾

用户希望一键将工作台分为「代码区」（资源管理器 + 编辑器）与「会话区」（会话列表 + 聊天），以便在两块物理屏幕上分别工作。

### 方案对比表

| 维度 | 提案 1 单窗口分屏模式 | 提案 2 双 BrowserWindow |
|------|----------------------|-------------------------|
| 核心思路 | 同一窗口内左右两大列 + 可拖拽中缝 | 主窗代码区 + 子窗仅会话区 |
| 主要改动范围 | `App.vue`、`TitleBar`、CSS | `electron/main`、preload、`App.vue`、进度广播 |
| 优点 | 改动小、无 IPC 双窗同步 | 真正双屏、符合用户「两个窗口」预期 |
| 缺点 / 风险 | 不能独立拖到副屏 | 双窗生命周期、进度事件广播 |
| 工作量（粗估） | 中 | 大 |
| 适合场景 | 超宽屏 / 单屏分区 | 双显示器 |

### 关键差异说明

- 选提案 1：实现快，但右栏无法单独放到第二块屏。
- 选提案 2：点击后变为两个独立 Electron 窗口，可拖到不同显示器。
- 两方案可分期：先 1 后 2；用户明确要双窗则必须 2。

### 推荐方案

**推荐：提案 1 – 单窗口分屏模式**（改动集中前端、风险低）。若用户强需求物理双屏，选提案 2。
