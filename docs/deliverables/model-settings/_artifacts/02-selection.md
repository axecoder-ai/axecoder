# 选型记录：model-settings

## 2a 选型摘要

### 一句话需求回顾

改善 AxeCoder 中「模型设置」的发现与使用体验：协作工坊在未配置模型时仅弹 `alert`、无模型选择器；设置页打开后默认不在「模型」Tab；需在设置中支持连通性检测与中文导航。

### 方案对比表

| 维度 | 提案 1 Chat 对齐 | 提案 2 最小修补 | 提案 3 增强 Models |
|------|------------------|-----------------|---------------------|
| 核心思路 | 工坊复用 ModelPicker + 深链 Models | 空状态条 + 深链，无下拉 | 在 1/2 上增加中文 Tab、测试连接、列表状态 |
| 主要改动范围 | WorkshopPane、SettingsPanel、App | WorkshopPane banner、Settings 深链 | 上述 + models:ping IPC、ModelsTab |
| 优点 | 与 Chat 一致、可换模型 | 改动最小 | 配置可验证、导航更友好 |
| 缺点 / 风险 | 共用 activeModelId | 工坊不能当场换模型 | 工作量中、真实 API 依赖 |
| 工作量 | 中 | 小 | 中–大 |
| 适合场景 | 默认推荐 | 只求去掉 alert | 要重视配置质量与可发现性 |

### 关键差异说明

- 提案 1/2 解决「工坊 alert + 找不到设置」；提案 3 额外解决「配完不知道通不通」。
- 提案 3 默认以提案 1 为工坊基线（内联选择器），否则测试连接仍无法在服务内选模型。
- 不重做 `models.json` 存储。

### 推荐方案

**推荐：提案 1**（工坊 UX）+ 按需叠加提案 3 的 ping。

**推荐理由：** 截图痛点首先是工坊无模型入口；提案 3 的 ping 与中文 Tab 对长期维护有价值，与现有 Models 存储兼容。
