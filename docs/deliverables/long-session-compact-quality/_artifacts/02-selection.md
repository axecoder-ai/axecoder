# 选型记录

## 一句话需求回顾

在已有 Agent LLM 摘要 compact 基础上，修复长会话多轮后质量下降（遗忘早期决策、文件路径、工具结论）；用户要求 **Chat `/compact` 斜杠同步升级为 LLM 摘要**。

## 方案对比表

| 维度 | 提案 1 先摘要后 FRC + 滚动摘要 | 提案 2 统一管线 + 结构化摘要 |
|------|-------------------------------|------------------------------|
| 核心思路 | 调换 FRC/compact 顺序；滚动摘要字段；`/compact` 接 LLM | 统一服务 + toolLog 补强 + 动态 keepTail |
| 主要改动范围 | agent-loop、context-compact、session-store、chat-compact、slash | 新服务模块、config、矩阵文档 |
| 优点 | 改动小；直击 FRC 竞态根因 | Chat/Agent 完全一致；摘要结构化 |
| 缺点 / 风险 | 滚动摘要过长需截断 | 工作量中；动态 keepTail 行为复杂 |
| 工作量 | 小 | 中 |
| 适合场景 | 快速修复长会话质量 | 需要统一管线与 Settings 开关 |

## 关键差异说明

- 选提案 1：先让 LLM 看到完整 tool 再 FRC 清理，多轮摘要可累积。
- 选提案 2：额外做 toolLog 注入与 cooldown，本期不做。
- 用户调整：**必须**升级 Chat `/compact` 为 LLM 摘要（提案 1 原文为可选）。

## 推荐方案

**推荐：提案 1 – 先摘要后 FRC + 滚动摘要沉淀**

## 用户最终选择

- **选定：** 提案 1 – 先摘要后 FRC + 滚动摘要沉淀
- **调整说明：** 必须同时升级 Chat `/compact` 为 LLM 摘要
