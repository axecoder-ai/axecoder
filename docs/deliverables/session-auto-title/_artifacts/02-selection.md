# 选型记录

## 2a 选型摘要

**需求回顾：** Agents 侧栏会话名应反映对话主题；当前仅首条用户消息 24 字截断，易出现「你好」等无主题标题。

| 维度 | 提案 1 LLM 标题 | 提案 2 启发式 | 提案 3 混合 |
|------|-----------------|---------------|-------------|
| 核心思路 | 每轮后 LLM 生成主题 | 关键词/规则提取 | 首句占位 + 多轮后 LLM 刷新 |
| 改动范围 | Main IPC + ChatPane | 启发式模块 | 同 1 + 条件触发 |
| 优点 | 主题准确 | 零 API 成本 | 首轮快、后续准 |
| 缺点 | 每轮 API | 质量不稳定 | 标题会闪变一次 |
| 工作量 | 中 | 小 | 中 |
| 适合场景 | 质量优先 | 离线/极简 | 平衡体验与成本 |

**推荐：** 提案 1（质量最佳）。

**关键差异：** 提案 3 保留首条截断占位，仅在 `messages.length >= 4` 且 title 仍为占位时触发一次 LLM，API 次数少于每轮刷新。

## 2b 用户选择

- **选定：** 提案 3 – 混合：首句截断 + 第二轮后 LLM 刷新
- **调整：** 本期不做用户手动重命名 UI，仅自动更新；非默认 title 不覆盖（隐含于方案）
