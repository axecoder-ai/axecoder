# 选型记录 – collab-workshop-display

## 2a 选型摘要

**需求回顾：** 协作 Workshop 中思考过程须出现在 AI 回复气泡下方；AI 身份来自 `~/.aex-coder/users.json` 不得编造；每条用户输入须完整保留。

| 维度 | 提案 1 复合消息 | 提案 2 UI 重排 |
|------|-----------------|----------------|
| 核心思路 | `reasoningContent` 附挂在同条 assistant 消息 | 前端合并 reasoning+summary 双条 |
| 改动范围 | 类型+编排+存储迁移+UI |  mainly 前端 |
| 优点 | 持久化与展示一致、跨端简单 | 改动小 |
| 缺点 | 需迁移历史 reasoning 条 | 合并规则易碎 |
| 工作量 | 中 | 小 |
| 适合 | 长期维护、严格身份 | 快速补丁 |

**推荐：提案 1** — 与「思考在回复下方」语义一致，避免双条漂移。

## 2b 用户选择

- **选定：** 提案 1 – 单条消息复合结构
- **调整：** 严格模式 — 无 `users.json` 绑定的员工角色不允许发言
