# 选型记录

## 2a 选型摘要

### 一句话需求回顾

允许多个独立 AxeCoder 进程（多开 App），每个打开不同项目；**同一项目路径禁止两进程同时打开**。

### 方案对比表

| 维度 | 提案 1 文件锁 | 提案 2 文件锁+套接字 |
|------|---------------|---------------------|
| 核心思路 | 去单实例 + userData 锁文件 | 同左 + 重复打开时聚焦已有进程 |
| 改动范围 | index、fs-ipc、锁模块 | 再加跨进程 socket |
| 优点 | 小、满足互斥 | 体验更好 |
| 缺点 | 重复打开仅提示 | 跨平台复杂 |
| 工作量 | 中 | 大 |

### 推荐

**推荐：提案 1** — 满足多进程与同项目互斥，改动最小。

### 用户选择（2b）

- **提案 1**
- **调整：** 必须 CLI 打开项目；菜单「新建窗口」；确认同项目绝不多开
