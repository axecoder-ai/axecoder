# 选型记录

## 一句话需求回顾

WebSearch（Serper）已接入但默认关闭，用户须先开开关再填 Key。本轮要降低启用摩擦：Settings 一键开，并支持环境变量配置。

## 方案对比表

| 维度 | 提案 1 Key 驱动自动启用 | 提案 2 Settings 一键开 |
|------|------------------------|------------------------|
| 核心思路 | 有 Key 即启用，去掉独立开关 | 保留开关，UX 合并为「启用→填 Key→保存即开」 |
| 主要改动范围 | config 派生逻辑、executor 门控、Settings 去 toggle | GeneralTab UX、保存时原子写 flag+key |
| 优点 | 步骤最少；env 友好 | 可关搜索但保留 Key；语义清晰 |
| 缺点 / 风险 | 难临时关闭 | 仍须 Settings 操作一次 |
| 工作量 | 中 | 小 |
| 适合场景 | 希望零开关、偏开发者 | 希望显式控制、产品化 Settings |

## 关键差异说明

- 选提案 1：填 Key 即生效，无法单独保留 Key 并关闭。
- 选提案 2：可一键关闭功能，Key 可保留；需点「启用」再填 Key。
- 两方案均可扩展 env；提案 1 更依赖 env，提案 2 以 UI 为主。

## 推荐方案

**推荐：提案 1 – Key 驱动自动启用**  
改动面集中在门控语义，长期与「有 provider 即用」一致。若需显式关闭搜索，选提案 2。

## 用户最终选择

- **选定提案：** 提案 2 – Settings 一键开（合并开关 UX）
- **调整说明：** 同时支持 `SERPER_API_KEY` 环境变量作为 API Key 来源
