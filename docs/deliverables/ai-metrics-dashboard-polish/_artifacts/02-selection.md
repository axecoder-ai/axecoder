# 方案选型记录

## 一句话需求回顾

在既有 AI 性能仪表盘（`AiMetricsPanel` + `ai-metrics-store`）上，提升文字对比度与视觉清晰度，并分阶段补齐指标展示与数据质量（真实 token、来源/Provider 维度、Trace 联动、时间范围等）。

## 方案对比表

| 维度 | 提案 1 视觉专项 | 提案 2 视觉+信息架构 | 提案 3 全量三期 |
|------|----------------|---------------------|----------------|
| 核心思路 | 仅 CSS + Canvas 图例 | 样式 + 表/图/语义扩展 | 提案 2 + usage/E2E/筛选/Trace/时间范围 |
| 主要改动范围 | `AiMetricsPanel.vue` | + `ai-metrics-store`、i18n、BottomPanel | + provider 层、Trace、设置 |
| 优点 | 快、低风险 | 看得清且读得懂 | 一次到位 |
| 缺点/风险 | 指标仍不全 | 无真实 token | 工作量大、provider 差异 |
| 工作量 | 小（2–3 人日） | 中（4–5 人日） | 大（10+ 人日） |
| 适合场景 | 急修可读性 | 平衡交付 | 长期演进 |

## 关键差异

- 提案 1 不动埋点与 IPC，用户仍看不到错误率曲线、来源维度。
- 提案 2 用已有字段补第三张图与错误率列，token 仍为估算。
- 提案 3 需改 provider 回传 usage，并做 Trace 联动与时间范围，回归面最大。

## 推荐方案

**推荐：提案 2 – 视觉 + 信息架构**

理由：在最小风险下同时解决「看不清」和「读不懂」；真实 token 与持久化可后续 PR 拆分。若用户强需求一次到位可选提案 3。

## 用户最终选择

- **选定：提案 3 – 全量三期路线图**
- **调整说明：** 无额外调整，按方案原文落地
