# 选型记录

## 2a 选型摘要

**需求回顾：** 实现可用的全局搜索——项目内文本搜索，入口清晰、体验对齐 VS Code；范围不含 CodeGraph 语义搜索。

| 维度 | 提案 1 – VS Code 轻量对齐 | 提案 2 – 完整搜索视图 |
|------|--------------------------|----------------------|
| 核心思路 | 顶栏改语义 + 侧栏输入即搜 + ⌘P | 提案 1 + 选项条 + glob + 替换 |
| 主要改动 | TitleBar、SearchPanel、QuickOpen | 再加 fs:search 扩展、replace IPC |
| 优点 | 改动小、见效快 | 功能完整对标 VS Code |
| 缺点/风险 | 无高级选项 | 工期大、替换写盘风险 |
| 工作量 | 小 | 大 |
| 适合场景 | 最小可用 | 完整搜索视图 |

**推荐方案：** 提案 1 – VS Code 轻量对齐（结合用户「只是要实现」与最小改动原则）。

**关键差异：** 提案 2 增加大小写/全词/正则、include/exclude glob、批量替换；后端需扩展 IPC。

## 2b 用户最终选择

- **选定提案：** 提案 2 – 完整 VS Code 搜索视图（选项条 + glob + 替换）
- **调整说明：** 无额外调整，按方案原文落地
